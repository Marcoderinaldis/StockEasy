# Report Evidence Map

This file maps development work to sections of the final university report.
Every significant piece of development should be recorded here with its report relevance.
The goal is to ensure the report can be written from complete evidence, not reconstructed from memory.

---

## Report sections and evidence

**Report section:** System requirements, user stories, functional and non-functional requirements
**Evidence sources:**

### Reflection

**Evidence sources:**
- docs/development-log/DEVELOPMENT_LOG.md (problems and fixes)
- docs/decision-log/DECISION_LOG.md (alternatives considered)

---

## Evidence collection rules

- Every completed feature must have at least one screenshot recorded in SCREENSHOT_CHECKLIST.md.
- Every architectural decision must have an entry in DECISION_LOG.md before the code is committed.
- Every development session must have an entry in DEVELOPMENT_LOG.md.
- Every test performed must have an entry in TESTING_LOG.md.
